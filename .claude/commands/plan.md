Read the plan file provided by the user and conduct a detailed interview to understand and refine the plan.

## Instructions

1. **First, ask the user for the plan file path** if not provided as an argument
2. **Read the plan file** using the Read tool
3. **Analyze the plan thoroughly** - understand its scope, goals, and components
4. **Conduct an in-depth interview** using AskUserQuestion tool to explore:

### Technical Implementation
- Architecture decisions and alternatives considered
- Technology choices and their rationale
- Integration points and dependencies
- Performance considerations
- Security implications
- Scalability approach

### UI & UX
- User flows and interactions
- Design decisions and user experience goals
- Accessibility considerations
- Mobile/responsive approach
- Error handling and feedback to users

### Concerns & Risks
- Potential blockers or challenges
- Technical debt implications
- Timeline and resource constraints
- Dependencies on external factors

### Tradeoffs
- What was prioritized vs deprioritized
- Build vs buy decisions
- Short-term vs long-term considerations
- Complexity vs simplicity choices

## Interview Style

- Ask one focused question at a time using AskUserQuestion
- Provide 2-4 meaningful options when applicable
- Follow up on interesting or unclear points
- Challenge assumptions constructively
- Help identify gaps or blind spots in the plan
- Summarize key decisions and insights as you go

## Output

After the interview, provide:
1. A summary of key decisions made
2. Any identified risks or concerns
3. Recommended next steps
4. Updated plan if significant changes were discussed
